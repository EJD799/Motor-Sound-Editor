# 项目详情与保存

查看和修改项目全局属性，以及 Motor Sound Editor 专属的 `.msep` 项目文件格式和工程的保存机制。

## 1. 项目详情设置

![项目信息](/docs/ProjectDetails.png)

项目详情包含了工程的基础元数据和全局模拟器参数。你可以在项目设置中修改以下核心信息：

* **基础信息**：
  * **项目名称**：当前工程的名称。
  * **文件路径**：文件存储的路径。
  * **音轨数**：当前音轨文件的条数。

* **模拟器参数**：
  这些参数决定了声音预览时的物理模拟表现形式 ：
  * **最高速度 (Max Speed)**：列车或电机的最大运行速度。
  * **加速度 (Acceleration)**：牵引（Traction）与制动（Brake）状态下的加速度。

## 2. MSEP 文件格式


Motor Sound Editor 采用独有的 `.msep` 格式作为项目工程文件。

* **一体化打包**：与许多依赖外部音频链接的软件不同，`.msep` 格式不仅包含了工程的基础配置文件，还会将你导入的音频资产（如 `wav`、`ogg`）的二进制数据直接嵌入并打包在文件内部 。
* **优势**：这意味着你在不同电脑之间分享 `.msep` 工程文件时，**不需要**连同音频源文件一起打包发送，音频不会发生“文件丢失”的问题。

## 3. 保存与未保存提示


* **未保存状态 (Dirty State)**：当你添加轨道、修改关键帧或调整项目详情时，软件会自动将其标记为“未保存”状态（Dirty）并更新修改时间。通常在标题栏的文件名旁会出现一个星号 `*`，提醒你当前有未保存的改动。
* **保存 (Save)**：保存操作会将内存中的文档结构和音频缓存同步写入到本地的 `.msep` 文件中 [cite: src/services/msepProject.ts]。
* **另存为 (Save As)**：你可以通过另存为创建一个全新的 `.msep` 文件副本。
* **撤销与重做**：保存操作通常与撤销/重做历史记录独立运行。即使保存了工程，你只要没有关闭软件，依然可以撤销近期的操作。
