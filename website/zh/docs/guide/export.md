# 导出流程

当您完成所有的音轨编辑和曲线参数调整后，就可以将当前工程导出为目标模拟器可用的插件包。

## 1. 导出前检查项

在进行导出操作之前，请确保您的工程满足以下条件：
* **有效音轨**：导出引擎只会处理那些**已启用 (Enabled)**、**未静音 (Unmuted)** 并且**已分配音频资产**的音轨。如果您有不希望导出的草稿音轨，将其静音或禁用即可。
* 如果所有导出轨都没有分配音频资源，导出操作将会报错提示。

## 2. 导出对话框

![项目导出](/docs/export.png)

点击菜单栏的导出按钮后，会弹出“导出文件 (Export File)”对话框，您可以在此配置最终输出的格式和音频参数。

## 3. 导出格式 

* **BVE**：目前软件首要支持的导出格式。选择此格式后，软件会将项目自动打包生成一个标准的 **ZIP 压缩包**（例如：`[项目名称]-BVE.zip`），其中包含所需的配置文件和音频。
* **OpenBVE / MTR**：这两种格式目前在菜单中处于禁用（占位）状态，将在未来的版本更新中开放支持。

## 4. 采样率 

您可以为导出的音频指定全局采样率。
* 可选的采样率包括：**22050 Hz, 32000 Hz, 44100 Hz, 48000 Hz, 96000 Hz**。
* 默认推荐值为 **44100 Hz**，这能在音质和文件大小之间取得良好的平衡。

## 5. 格式转换与自动适配


Motor Sound Editor 原生支持导入 `.ogg` 和 `.wav` 等格式的音频源文件。为了确保最终导出的插件包能在目标模拟器中完美运行，软件内置了**智能音频格式转换**功能。

当您在音轨中使用的音频格式与所选的导出标准不完全匹配时（例如：标准 BVE 引擎仅支持 WAV 格式，而您的工程中包含了 OGG 音频），对话框底部会自动显示一条黄色的提示信息：
> *"Warn ogg file: OGG audio will be converted to WAV for BVE export."*

**您无需手动干预，也无需借助第三方音频软件进行转码。**
在最终导出打包时，Motor Sound Editor 会根据您选择的导出目标（无论是当前的 BVE 格式，还是后续支持的 OpenBVE、MTR 等任何其他标准），**自动将所有不兼容的音频无缝转换为该模拟器所要求的合规格式**。配置确认无误后，点击右下角的 **Export** 按钮即可安心完成导出。
